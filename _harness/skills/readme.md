# SKILLS

This directory defines the harness skill format used by the template.

## Standard

A harness skill should follow:

- frontmatter for routing metadata
- markdown body for execution protocol

## Maintainer guidance

When evolving the skill system:

- keep routing metadata minimal
- keep the protocol body operational
- do not duplicate global rules inside every skill
- use skills for capability units, not for generic project description
