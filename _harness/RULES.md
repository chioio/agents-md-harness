# RULES

## Core rules
- Keep the root `AGENTS.md` thin.
- Do not duplicate the same instructions across files unless necessary.
- Prefer explicit routing over implicit assumptions.
- Prefer minimum necessary loading over reading everything.
- Keep files modular by responsibility.

## Content boundaries
- `README.md` explains what the harness is.
- `ROUTING.md` explains how to choose files.
- `CATALOG.md` explains where things are.
- `WORKFLOW.md` explains how to act.

## Non-goals for v1
- No full schema yet.
- No automatic parser requirement yet.
- No deep module hierarchy yet.
