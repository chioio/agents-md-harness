# ROUTING

先按任务路由，再继续加载更多文件。

## 任务到文件映射

- 了解全栈项目目标与协作方式 -> `readme.md`
- 查找 FE / BE / harness 文件位置 -> `catalog.md`
- 确认边界、审批条件、文档要求 -> `rules.md`
- 执行产品 / 架构 / 开发 / 测试 / UI/UX 工作流 -> `workflow.md`
- 查看长期决策、规范、失败案例、测试结论 -> `memory/project.md`

## 常见任务模式

- 产品文档输出 -> `readme.md`, `workflow.md`, `rules.md`
- 架构设计 / 架构文档 -> `workflow.md`, `rules.md`, `memory/project.md`
- FE 实现 -> `catalog.md`, `rules.md`, `workflow.md`
- BE 实现 -> `catalog.md`, `rules.md`, `workflow.md`
- FE / BE API 对接 -> `catalog.md`, `rules.md`, `workflow.md`, `memory/project.md`
- 测试设计 / 回归验证 -> `workflow.md`, `catalog.md`, `memory/project.md`
- UI / UX 方案与实现 -> `workflow.md`, `rules.md`, `catalog.md`

## 原则

只加载当前任务真正需要的最小文档集合。
