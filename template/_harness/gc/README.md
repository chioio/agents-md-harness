# Harness GC

Harness GC is responsible for keeping the harness small, relevant, and maintainable over time.

It applies to:

- short-term per-agent memory
- long-term project memory
- outdated or duplicated harness content

## Goals

- remove stale working context
- compress repeated notes
- promote only durable knowledge
- keep the harness readable and useful
